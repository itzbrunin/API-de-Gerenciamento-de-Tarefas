🚀 Push para GitHub

SEM SSH - Usando HTTPS com Personal Access Token

⏰ Antes de começar: Você vai precisar de um Personal Access Token (PAT) do GitHub. Demora 2 minutos para gerar.

🔑 Passo 1: Gerar seu Personal Access Token

1. Acesse https://github.com/settings/tokens/new
2. Preencha:
  • Note: API-Tarefas-Push
  • Expiration: 30 dias (ou mais)
  • Scopes: Marque apenas ☑️ repo
3. Clique em "Generate token"
4. COPIE o token! (Formato: ghp_xxxxxxxxxxxxx)
⚠️ Você não consegue ver novamente, então copie AGORA!

🎯 Passo 2: Escolha uma opção para fazer o Push

1

Script Automático (MAIS FÁCIL)

Abra o PowerShell
Navegue para a pasta do projeto:
cd "D:\Pastas de Programação treinamentos\ProjetoAPI\task_api"
Execute o script:
.\push-with-token.ps1
Cole o token quando pedir
2

Windows Credential Manager (RECOMENDADO)

Abra o PowerShell e navegue para:
cd "D:\Pastas de Programação treinamentos\ProjetoAPI\task_api"
Execute:
git push -u origin main
Uma janela pedirá autenticação:
Username: itzbrunin
Password: [Cole seu token aqui]
Salvar credenciais
✅ Mais seguro: Token não fica visível no histórico do PowerShell
3

Comando Manual (Mais Controle)

Abra o PowerShell:
$token = "ghp_xxxxxxxxxxxxx" # Substitua pelo seu token cd "D:\Pastas de Programação treinamentos\ProjetoAPI\task_api" git remote set-url origin "https://itzbrunin:$token@github.com/itzbrunin/API-de-Gerenciamento-de-Tarefas.git" git push -u origin main
E pronto! Seu projeto está no GitHub.
⚠️ Token fica visível na command line (menos seguro)

⭐ Recomendação

Use a Opção 2 (Windows Credential Manager) porque:

  • ✅ Token não fica visível na command line
  • ✅ Windows salva as credenciais para futuras operações
  • ✅ Funciona com qualquer ferramenta Git
  • ✅ Reutilizável (não precisa toda vez)
